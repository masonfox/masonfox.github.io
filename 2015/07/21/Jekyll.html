Jekyll

July 21, 2015

I guess this marks my first “real” post! To start this off, I’d love to talk about Jekyll, something I have become very passionate about within the past couple of weeks.

What is Jekyll?

Jekyll is a static site generator built on Ruby and also utilizing liquid templating. Jekyll is not a server side client, but rather local scripts that compiles the file types into a single static directory. It utilizes liquid templating, which allows users to designate if/else statements, declare specific html layouts, and more.

One of the most important concepts of Jekyll is the content and structure separation. As explained briefly earlier, Jekyll keeps the content of the website and the strucutre/layout of the website in separate directories. An example of this would be this blog post. This post is located in the posts directory, which at this moment, is written in markdown. By adding front matter to the top of the post, triggered by three dashes, one is able to choose a specific layout that coincides with the desired structure of the content. For example, I am writing a post, a Jekyll blog post, so I would note in my front matter in my markdown file that I would like my layout for this post to be, well, a post! One is also able to add additional information, such as the time code, the title of the post, and excerpt of the post, an image, and much more! Pages are created in the same exact way, but knowledge of html, css, and liquid templating are required to ensure correct Jekyll builds.

Pros of Jekyll

  1. Static sites run faster as there is no database weighing it down. They are solely comprised of html documents and their assets.
  2. Easy authoring: written in markdown, so the barrier to entry for new users is very simple. A blog post ui was also created and can be linked to your Github Repository at Prose.
  3. Hosted on github pages for free! (This one is)
  4. Simple build and deploy: as simple as running jekyll s (for serve to view the site locally) and jekyll b (for build to compile the directory) in the command line.
  5. Dynamic capabilities are possible for static pages, such as An example may be Google Forms to send form data, Disquis comment system and more API’s and embed’s.
  6. Supports many frameworks such as Twitter Bootstrap, Foundation 5, and more.
  7. Task runner support, such as Grunt or Gulp.

Cons of Jekyll

  1. Not officially supported on Windows.
  2. Does not have a database, meaning that it may be the best solution for every project.

I definitely recommend the going and trying out Jekyll. I have linked a couple videos down here so you can get some information about it!
